# Nina 2.0 Design

## Critical fixes (release gate)
- `auth/config.js`: replace base64 "encryption" with keytar (OS keychain), fallback AES-256-GCM + scrypt key. Migrate existing credentials.json on first run.
- `core/shell.js`, `plugins/system.js`: replace string-interpolated exec with `execFile`/`spawn` argv arrays. No shell string concatenation.
- `auth/oauth.js`: generate + validate OAuth `state` param to close CSRF gap.
- `core/agents.js`: add lock around worker queue `shift()` to fix race condition.

## 1. Provider тАФ tool-calling + UX
- Native tool-calling schema (write_file/run_cmd/delete_file) for Anthropic/OpenAI/Gemini in `providers/index.js`. Regex-based action parser (`core/executor.js`) stays as fallback for non-tool-calling providers.
- `auth/web-auth.js`: auto-`/close` the localhost page after successful `setActive()` (no manual click needed).
- Ollama model dropdown: on provider select, call `/test-ollama`, populate `model-select` with real installed models (replaces empty static list).

## 2. Team mode тАФ shared blackboard
- Shared state `{findings:[], status:{}}` in `core/agents.js`. Each agent appends events (role, action, result); other agents/Reviewer read it.
- Live Monitor renders blackboard events as they arrive.
- Protected by same lock as queue fix above.

## 3. DX тАФ plugins + approval flow
- `/plugins` command: list/enable/disable plugins.
- Load third-party plugins from `~/.nina/plugins/` via dynamic `import()`, common `{name, commands, hooks}` interface.
- Approval modes via `/system`: `always` / `risky-only` / `never`. `risky-only` requires confirm for RUN_CMD/DELETE_FILE; WRITE_FILE shows diff (`core/diff.js`) inline.

## 4. Security тАФ sandboxed execution
- Denylist regex for destructive commands (`rm -rf`, `del /s`, `format`, `shutdown`, etc.) тАФ always requires confirmation even in autorun.
- Optional `--sandbox` flag: run RUN_CMD inside Docker container, cwd mounted, network off by default.
- Ties into approval flow's `risky-only` mode.

## Testing notes
- Tool-calling: mock tool_call responses per provider + fallback path for non-tool-calling providers.
- Blackboard: 3-agent sim (plannerтЖТcoderтЖТreviewer), verify reviewer feedback reaches coder.
- Approval flow: mock action lists, approve/reject paths.
- Sandbox: denylist regex unit tests; Docker mode smoke test (`echo` in container).
